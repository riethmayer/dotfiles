# Part 4: Execute and Measure

Based on the RDCL Toolkit v3.1.0 (CC BY-SA 4.0) by radicalproduct.com.

## Instructions

Translate your "Now" milestones from the Strategic Roadmap into measurable **metrics**, **hypotheses**, and **tasks**. This is the final step toward an actionable, vision-driven product plan.

Use one Execute & Measure Plan per Initiative. Each should be assigned to a responsible individual or team.

### Key Metrics
How will we know the milestone has been achieved? What specific, concrete values can we measure?

**Favor leading metrics over trailing ones** — seek measurements that show success or failure as early as possible, so you can test your hypotheses and adjust your planned activities if necessary.

### Hypotheses
What activities or achievements do we think will drive the metrics? Structure as:

```
If... (activity)
Then... (outcome/metric change)
Because... (cause or connection — why we believe this)
```

Hypotheses must be **falsifiable** — you must be able to actually measure whether the "Then" happened.

### Tasks
Break activities from hypotheses into concrete, actionable tasks. These can be captured in project management software (Linear, Trello, JIRA, etc.).

## Important: Don't work in a vacuum

The Execute & Measure Plan is the most important exercise to do **collaboratively**. You need domain-specific implementation knowledge to understand:
- How certain activities will achieve strategy goals
- Which leading metrics are most likely to predict success

## Output format

For each "Now" initiative:

```markdown
## Execute & Measure: {Initiative Name}

**Responsible:** {team/person}
**"Now" Milestone:** {milestone from roadmap}

### Key Metrics
- {metric 1}: {target value or threshold}
- {metric 2}: {target value or threshold}

### Hypotheses

| If... (Activity) | Then... (Outcome) | Because... (Reason) |
|---|---|---|
| {activity} | {measurable outcome} | {causal reasoning} |
| ... | ... | ... |

### Tasks
- [ ] {task 1}
- [ ] {task 2}
- [ ] ...
```

## Revisit cadence
- Monthly or more often
- Each revisit: check metrics, validate/invalidate hypotheses, adjust tasks
