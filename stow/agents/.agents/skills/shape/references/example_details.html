Scope, capabilities, lifecycle canvas, and open decisions for the v1 of skills.earlybird.com + the ebsk CLI. Read the pitch first if you haven't.

Problem

Skills and prompts that materially improve our work live on individual laptops with no path to sharing, vetting, or maintenance. The bigger miss is that we don't have a shared layer for how Earlybird actually works — the workflows that move the needle aren't codified, so onboarding stays manual and the team can't compound on its own conventions.

Generic catalogs can't fix this. Skills get their value from referencing our tables, our stages, our jargon. Once they reference Earlybird specifics, distribution + safety + maintenance become the actual product. That's what we're shipping.

Scope
In
  • Open-source CLI (ebsk) + Cloud Run registry + Next.js web library, Apache 2.0
  • Earlybird as tenant zero on skills.earlybird.com
  • Manual blessing workflow (engineering admin reviews PRs against earlybirdvc/skills)
  • Two install tracks: CLI sync (Claude Code · Codex) and manual zip (ChatGPT · Perplexity)
  • Opt-in cloud backup for personal namespace + ebsk share preview URLs
Out (roadmap)
  • Zero-knowledge architecture (operator-readable for now)
  • Multi-tenant SaaS sign-up flow — other firms self-host until hosted opens
  • Full LLM-judge eval framework wired into CI (manual eval for MVP)
  • Chrome extension that injects prompts into chatgpt.com
  • OpenAI Custom GPT API auto-provisioning
  • Cross-workspace subscription / syndication (workspace-scoped v1)
Success metric
Single measurable outcome
≥ 15 blessed skills · ≥ 8 authors across ≥ 3 departments, by end of Q3 2026
Coverage signals that the library has crossed from "engineering's project" to "how the org works." Depth (per-skill usage) and velocity (time-to-bless) become KRs in Q4 once we have the volume to measure them.
Capabilities

Verb-noun cards. Each maps to one or more CLI commands or web surfaces; modalities (cli / web / sync-policy) tell you where they live.

install-skill
Pull a blessed skill from the registry into the local skill loader (Claude Code / Codex / manual zip for others).
Domain: distributionModalities: cli · web · cron-sync
author-skill
Scaffold and iterate locally with ebsk init. Opt-in private cloud backup; share via non-guessable preview URL on demand.
Domain: authoringModalities: cli
submit-skill
Conscious-gate promotion from personal → workspace / public. Opens a PR against earlybirdvc/skills with the blessing checklist.
Domain: authoringModalities: cli
bless-skill
Admin merges the blessing PR after hooks / PII / eval / visibility checks. Blessing = deployment.
Domain: governanceModalities: cli · gh
fork-skill
Personal copy with auto-update tracking; checksum-tracked divergence pins the fork at the diverged version.
Domain: authoringModalities: cli
unbless-skill
Admin removes a blessed skill. Next ebsk sync on every workspace laptop deletes it; users prompted to fork before sync removes the file.
Domain: governanceModalities: cli · cron-sync

Auth scope for non-Earlybird users
No-auth on public skills + earlybird.com-gated OAuth for workspace skills · alternatives below
Ship-gate

Public path: no auth. Workspace path: Google OAuth restricted to @earlybird.com (mirrors the ee-webapp-auth pattern). Alternative: loosen OAuth to any Google identity. Alternative: shared passcode handed out by admin — lightweight but theatrical.
